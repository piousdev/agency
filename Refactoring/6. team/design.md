## 1. Design Overview

### 1.1 Purpose of This Document

This document describes the structural design, layout, and feature organization
of the Team navigation without specifying visual styling, colors, or specific
implementation details.

### 1.2 Design Principles

- **Clarity**: Quick access to team information
- **Transparency**: Clear visibility into capacity and workload
- **Efficiency**: Streamlined processes for common actions
- **Privacy**: Appropriate access controls for sensitive data
- **Support**: Enable effective team management and collaboration

- All dashboard widgets visible

### 10.2 Tablet (768px-1919px)

- Simplified capacity view

## 11. Accessibility Features

### 11.1 Keyboard Navigation

- Tab through all team members
- Arrow keys to navigate
- Enter to open profile
- Shortcuts: N (new member), / (search), F (filter)

### 11.2 Screen Reader Support

- ARIA labels on all interactive elements
- Status changes announced
- Capacity indicators described
- Time-off status announced

### 11.3 Visual Accessibility

- Status not solely by color
- Text labels on all indicators
- High contrast mode support
- 200% zoom support
