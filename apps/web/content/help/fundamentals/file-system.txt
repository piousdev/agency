# File system

The API supports multiple response formats depending on your needs.

## JSON (default)

Standard JSON format with camelCase field names:

```json
{
  "success": true,
  "data": {
    "userId": "123",
    "userName": "John Doe",
    "createdAt": "2024-01-01T00:00:00Z"
  }
}
```

## Snake case

Some legacy endpoints support snake_case format via the `X-Case-Style` header:

```json
{
  "success": true,
  "data": {
    "user_id": "123",
    "user_name": "John Doe",
    "created_at": "2024-01-01T00:00:00Z"
  }
}
```

## Content negotiation

You can request different formats by setting the `Accept` header:

- `application/json` - Standard JSON (default)
- `application/xml` - XML format
- `text/csv` - CSV format (for list endpoints)

## File uploads

For file uploads, use `multipart/form-data`:

```
POST /api/files
Content-Type: multipart/form-data

file: [binary data]
metadata: {"name": "document.pdf"}
```

## Best practices

1. Always specify the `Content-Type` header in requests
2. Use `Accept` header to request specific formats
3. Handle compression with `Accept-Encoding: gzip`
4. Set reasonable timeout values for large file operations
