# Alternative Schemas

## File system

The API supports multiple response formats depending on your needs.

### JSON (default)

Standard JSON format with camelCase field names:

```json
{
  "success": true,
  "data": {
    "userId": "123",
    "userName": "John Doe",
    "createdAt": "2024-01-01T00:00:00Z"
  }
}
```

### Snake case

Some legacy endpoints support snake_case format via the `X-Case-Style` header:

```json
{
  "success": true,
  "data": {
    "user_id": "123",
    "user_name": "John Doe",
    "created_at": "2024-01-01T00:00:00Z"
  }
}
```

## Describing responses

### Success responses

All successful API responses follow this structure:

```json
{
  "success": true,
  "data": {
    /* response data */
  },
  "message": "Optional success message"
}
```

### Error responses

Error responses include details about what went wrong:

```json
{
  "success": false,
  "message": "Error description",
  "errors": [
    {
      "field": "email",
      "message": "Invalid email format"
    }
  ]
}
```

### Paginated responses

List endpoints return paginated data with metadata:

```json
{
  "success": true,
  "data": [
    /* array of items */
  ],
  "pagination": {
    "page": 1,
    "pageSize": 20,
    "totalCount": 100,
    "totalPages": 5,
    "hasNextPage": true,
    "hasPreviousPage": false
  }
}
```
